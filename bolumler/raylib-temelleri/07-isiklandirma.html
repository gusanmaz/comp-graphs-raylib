Neden Işık Lazım?

Işık olmadan 3D nesneler düz görünür — sanki 2D şekiller gibi. Bir küre, ışıksız çizildiğinde düz bir daire gibi görünür. Işık eklediğinizde derinlik, hacim ve gerçekçilik kazanır.

Gerçek dünyada gördüğümüz her şey şu zincirin sonucudur:

  1. Işık kaynağı ışık yayar (güneş, ampul)
  2. Işık nesneye çarpar
  3. Nesne yüzeyinden yansıyan ışık göze ulaşır

Normal Vektör — Işığın Temeli

Normal = yüzeyin "dışarı bakan yönü". Düz bir duvarın normali duvara dik olan oktur.

  • Küre normali: normalize(nokta - merkez) — her noktada farklı yöne bakar
  • Kutu normali: her yüzün sabit bir normali var:
    • Üst yüz: (0, 1, 0) — yukarı bakar
    • Sağ yüz: (1, 0, 0) — sağa bakar
    • Ön yüz: (0, 0, -1) — öne bakar

Lambert Diffuse Modeli

En basit ışık modeli. Formül:

yogunluk = max(0, dot(N, L))

Burada:

  • N = yüzey normali (birim vektör)
  • L = ışığa doğru yön (birim vektör)
  • dot(N, L) = iç çarpım = iki vektörün ne kadar "aynı yöne" baktığı

dot product Ne Hesaplar?

0.0N ve L dik açıdaAydınlık yok
< 0Işık arka taraftan geliyorAydınlık yok (0'a clamp)
// dot product hesabi
float dot = N.x*L.x + N.y*L.y + N.z*L.z;

// Raylib'de hazir fonksiyon:
float dot = Vector3DotProduct(N, L);

Ambient + Diffuse

sonucRenk = temelRenk * (ambient + diffuse)
  • ambient = 0.1 — sabit minimum ışık. Olmasa tamamen karanlık yüzler oluşur.
  • diffuse = Lambert hesabı — yüzeyin ışığa ne kadar baktığına göre.

Renk Hesabı

Color skalaRenk(Color c, float k) {
    if (k > 1.0f) k = 1.0f;  // tasmasin
    return (Color){
        (unsigned char)(c.r * k),
        (unsigned char)(c.g * k),
        (unsigned char)(c.b * k),
        255
    };
}

Rengin RGB değerlerini yoğunlukla çarpıyoruz. k = 0 → siyah, k = 1 → orijinal renk.

Örnek Kod: Lambert Işıklandırma

🎮 Raylib: ornek13_isiklandirma.c

WASD ile ışık kaynağını taşıyın. Nesnelerin aydınlanmasının nasıl değiştiğini gözlemleyin. SPACE ile ambient'ı kapatın.

#include "raylib.h"
#include "raymath.h"

float lambert(Vector3 N, Vector3 L)
{
    float d = Vector3DotProduct(N, L);
    return d > 0.0f ? d : 0.0f;
}

Color skalaRenk(Color c, float k)
{
    if (k > 1.0f) k = 1.0f;
    return (Color){ (unsigned char)(c.r*k),
                    (unsigned char)(c.g*k),
                    (unsigned char)(c.b*k), 255 };
}

int main(void)
{
    const int W = 950, H = 650;
    InitWindow(W, H, "Ornek 13 - Isiklandirma");
    SetTargetFPS(60);

    Camera3D kam = { {6,6,6}, {0,0,0}, {0,1,0}, 50, CAMERA_PERSPECTIVE };
    Vector3 isikPos = { 3, 4, 2 };
    Color isikRenk = WHITE;
    float ambient = 0.1f;
    bool ambOn = true;

    // Kutu yuzlerinin normal vektorleri
    Vector3 kutuN[6] = {
        {0,0,-1},  // on
        {0,0,1},   // arka
        {-1,0,0},  // sol
        {1,0,0},   // sag
        {0,1,0},   // ust
        {0,-1,0},  // alt
    };

    while (!WindowShouldClose())
    {
        UpdateCamera(&kam, CAMERA_ORBITAL);

        // Isik kaynagini tasi
        if (IsKeyDown(KEY_D)) isikPos.x += 0.1f;
        if (IsKeyDown(KEY_A)) isikPos.x -= 0.1f;
        if (IsKeyDown(KEY_W)) isikPos.z -= 0.1f;
        if (IsKeyDown(KEY_S)) isikPos.z += 0.1f;
        if (IsKeyDown(KEY_UP))   isikPos.y += 0.1f;
        if (IsKeyDown(KEY_DOWN)) isikPos.y -= 0.1f;

        // Ambient toggle
        if (IsKeyPressed(KEY_SPACE)) ambOn = !ambOn;
        ambient = ambOn ? 0.1f : 0.0f;

        // Isik rengi
        if (IsKeyPressed(KEY_ONE))   isikRenk = WHITE;
        if (IsKeyPressed(KEY_TWO))   isikRenk = YELLOW;
        if (IsKeyPressed(KEY_THREE)) isikRenk = SKYBLUE;

        // Isik yonu hesapla
        Vector3 kutuM = { 0, 0.75f, 0 };
        Vector3 Lk = Vector3Normalize(Vector3Subtract(isikPos, kutuM));

        BeginDrawing();
            ClearBackground((Color){30,30,40,255});
            BeginMode3D(kam);
                DrawGrid(10, 1.0f);
                DrawSphere(isikPos, 0.15f, isikRenk);  // isik gorseli

                // Kutu: her yuz farkli aydinlikta
                float b = 1.5f, y = b/2;
                for (int i = 0; i < 6; i++) {
                    float lam = lambert(kutuN[i], Lk);
                    Color c = skalaRenk(RED, ambient + lam * 0.9f);
                    Vector3 p = Vector3Add(kutuM, Vector3Scale(kutuN[i], y));
                    DrawCube(p,
                        kutuN[i].x!=0 ? 0.02f : b-0.01f,
                        kutuN[i].y!=0 ? 0.02f : b-0.01f,
                        kutuN[i].z!=0 ? 0.02f : b-0.01f, c);
                }
                DrawCubeWires(kutuM, b,b,b, Fade(WHITE, 0.3f));

                // Normal vektorleri gorsellestir
                for (int i = 0; i < 6; i++) {
                    Vector3 s = Vector3Add(kutuM, Vector3Scale(kutuN[i], y));
                    DrawLine3D(s, Vector3Add(s, Vector3Scale(kutuN[i], 0.5f)), YELLOW);
                }
            EndMode3D();

            // UI
            DrawRectangle(0,0,W,50, Fade(BLACK,0.8f));
            DrawText("Isiklandirma Temelleri - Lambert Diffuse", 15, 13, 20, WHITE);
            DrawText("WASD:isik tasi  SPACE:ambient  1/2/3:renk", 15, H-25, 13, GRAY);
            DrawText(TextFormat("ambient=%.1f  lambert=max(0, dot(N,L))", ambient), 15, 60, 14, LIGHTGRAY);
        EndDrawing();
    }
    CloseWindow();
    return 0;
}

Satır Satır Açıklama

float lambert(Vector3 N, Vector3 L)

İki vektörün iç çarpımını hesaplar. Negatifse 0 döndürür (arka taraftaki yüzler karanlık kalır, negatif ışık olmaz).

Vector3Normalize(Vector3Subtract(isikPos, kutuM))

Işık yönünü hesaplıyor:
1. Subtract: kutu merkezinden ışığa doğru vektör
2. Normalize: uzunluğu 1 yapır (birim vektör) — sadece yön bilgisi kalır

Vector3Scale(kutuN[i], y)

Normal vektörü kutu yarı boyutu ile çarpar — yüzeyin merkezini bulur. Örn: üst yüz normali (0,1,0) × 0.75 = (0, 0.75, 0) → kutunun üstü.

Her yüze farklı DrawCube

Her yüzü ayrı ayrı çiziyoruz çünkü her birinin farklı rengi var. Normal yönüne dik olmayan boyutları 0.02f (çok ince) yaparak "düz panel" etkisi elde ediyoruz.

Shader Kavramı (Giriş)

Bu örnekte ışık hesabını CPU'da (C kodunda) yaptık. Gerçek oyunlarda bu hesap GPU'da yapılır — çok daha hızlı. GPU'da çalışan küçük programlara shader denir.

  • Vertex Shader: 3D koordinatları 2D ekran koordinatına dönüştürür
  • Fragment Shader: Her pikselin rengini hesaplar (ışık dahil)

Raylib'de shader kullanımı:

Shader shader = LoadShader("vertex.vs", "fragment.fs");
// ... kullan ...
UnloadShader(shader);

Raylib örneklerinde rlights.h başlık dosyası hazır ışık sistemi sunar. Bu ileri bir konudur — ama temeli burada öğrendiniz!

🧪 Deneyin: Ambient Kapatın

SPACE tuşuna basarak ambient'ı 0 yapın. Işığa bakmayan yüzler tamamen siyah olur — gerçek hayatta bunun olmamasının sebebi dolaylı ışık (indirect light) ve ortam yansımalarıdır.

🧪 Deneyin: Işığı Tepeden Getirin

UP tuşuna basarak isikPos.y'yi yükseltin (10+). Işık tam tepeden gelince sadece üst yüz aydınlanır — diğerleri karanlık.

🧪 Deneyin: Lambert Kontrolünü Kaldırın

d > 0.0f ? d : 0.0f yerine sadece return d; yazın. Negatif değerler rengi siyahın altına çeker — garip sonuçlar!

🧪 Deneyin: Işık Rengi

1/2/3 tuşları ile ışık rengini değiştirin. Sarı ışık altında kırmızı kutu turuncu görünür — renk karışımı!
